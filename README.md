# 🎮 Sales Quest

ระบบเกมเควสต์รายวัน ให้ทีมเซลหาลูกค้าใหม่ทุกวันแบบไม่หยุดหย่อน

ติดตั้งลงมือถือได้เหมือนแอปจริง (PWA) ไม่ต้องลง npm ไม่ต้องรัน server ไม่ต้องเสียค่าแอปสโตร์

---

## เริ่มใช้ใน 30 วินาที

ดับเบิลคลิก `index.html` → เลือกโปรไฟล์ → กดเควสต์ได้เลย

ข้อมูลเก็บใน localStorage ของเครื่องนั้น ถ้าอยากให้เซล 3 คนเห็นข้อมูลเดียวกัน ต้องต่อ Google Sheet (ด้านล่าง)

> ⚠️ เปิดจากไฟล์ตรงๆ (`file://`) จะ**ยังไม่ได้ฟีเจอร์แอป** (ไอคอน/ออฟไลน์) เพราะเบราว์เซอร์บังคับว่าต้องรันผ่าน http/https — ต้องเอาขึ้นเว็บก่อน (ดูหัวข้อถัดไป)

## ติดตั้งเป็นแอปบนมือถือ (PWA)

พอเอาขึ้นเว็บแล้ว เปิดลิงก์ในมือถือ จะมีแถบ **"ติดตั้งลงหน้าจอโฮม"** เด้งขึ้นมาให้กด

ได้ครบเหมือนแอปจากสโตร์:
- 📱 **ไอคอนโลโก้ MK** บนหน้าจอมือถือ
- 🖥️ เปิด**เต็มจอ ไม่มีแถบเบราว์เซอร์**
- ✈️ **ใช้ได้ตอนไม่มีเน็ต** — กดเควสต์ได้ปกติ ข้อมูลเข้าคิวไว้ พอเน็ตกลับมาส่งขึ้น Sheet เอง
- 🚀 อัปเดตทันทีที่แก้ไฟล์ ไม่ต้องรอรีวิวจากสโตร์

**iPhone:** ต้องเปิดด้วย **Safari** เท่านั้น → ปุ่มแชร์ ⎋ → "เพิ่มไปยังหน้าจอโฮม"
(เปิดจากใน Line หรือ Chrome จะไม่มีเมนูนี้ — แอปจะขึ้นวิธีทำให้อัตโนมัติ)

**Android:** กดปุ่ม "ติดตั้ง" ในแถบที่เด้งขึ้นมาได้เลย

### ไอคอน
สร้างจากโลโก้บริษัทต้นฉบับความละเอียดสูง
`F:\TUMCHZ WORK\MK\MK FOR WEB\Logo\MK LOGO\MK.png` (2000×2000)

ตัดเอาเฉพาะ **เพชร MK** — ไม่เอาแถบ "INDUSTRIAL SUPPLY CO., LTD" เพราะย่อเป็นไอคอนมือถือแล้วอ่านไม่ออก กลายเป็นรอยเปื้อน

ถ้าอยากสร้างใหม่ (เช่น เปลี่ยนโลโก้) รันสคริปต์นี้:
```
python make_icons.py
```
แก้ตัวแปร `SRC` (ไฟล์ต้นฉบับ) และ `DIAMOND` (กรอบครอป) ในสคริปต์ได้

| ไฟล์ | ใช้ตอน |
|------|--------|
| `icon-192.png` / `icon-512.png` | ไอคอนหลัก Android + หัวแอป |
| `icon-maskable-*.png` | Android ที่ครอปไอคอนเป็นวงกลม (เผื่อขอบไว้แล้ว) |
| `apple-touch-icon.png` | iPhone / iPad |
| `favicon-32.png` | แท็บเบราว์เซอร์ |
| `logo-full.png` | โลโก้เต็มพร้อมชื่อบริษัท (ไว้ใช้ที่อื่น) |

## รหัสผ่าน 🔐

เปิดแอปครั้งแรกจะให้ **ตั้งรหัสผ่าน** (อย่างน้อย 4 ตัว) หลังจากนั้นเครื่องนั้นจำไว้ ไม่ต้องใส่ซ้ำ

- เปลี่ยนรหัส / ล็อกเครื่องทันที → แท็บ **ตั้งค่า → ความปลอดภัย**
- ลืมรหัส → กด "ลืมรหัสผ่าน?" ที่หน้าล็อก (ล้างข้อมูลในเครื่องแล้วตั้งใหม่ — ถ้าต่อ Google Sheet ไว้ ข้อมูลยังอยู่ครบ ซิงก์กลับมาได้)
- รหัสเก็บเป็น **SHA-256 hash** ไม่ได้เก็บเป็นตัวหนังสือ

> ⚠️ **ขอบเขตของการล็อกนี้:** เป็นการล็อกฝั่งหน้าเว็บ กันคนทั่วไปที่ได้ลิงก์ไปเปิดดูข้อมูลลูกค้า/ยอดขายได้ **แต่ไม่ได้กันคนที่เก่งเทคนิคจริงๆ** (เปิด DevTools อ่านข้อมูลใน localStorage ได้) ถ้าต้องการความปลอดภัยระดับนั้นต้องมี server ตรวจรหัส ซึ่งเกินความจำเป็นสำหรับงานนี้

## ตามผลลูกค้า (สำคัญที่สุด)

แค่นับว่า "ทักไปกี่เจ้า" ไม่พอ — ต้องรู้ด้วยว่าแต่ละเจ้า**ไปจบที่ไหน** ไม่งั้นเซลกดมั่วให้ครบ 3 ทุกวันก็ได้แต้มเต็ม

### ขั้นตอนการขาย
ลูกค้า 1 เจ้าเดินผ่านขั้นพวกนี้:

| ขั้น | ความหมาย |
|------|-----------|
| 🆕 ทักแล้ว รอผล | เพิ่งทักไป ยังไม่รู้ผล (ตั้งต้นอัตโนมัติ) |
| 💬 ได้คุยแล้ว | คุยกับลูกค้าได้จริง |
| 🏭 ได้เข้าโรงงาน | ได้เข้าไปพบถึงที่ |
| 🧪 ส่งตัวอย่าง/ทดสอบ | ลูกค้ากำลังลองสินค้า |
| 💰 เสนอราคาแล้ว | รอคำตอบ |
| ✅ ปิดได้ | ได้ลูกค้า (ใส่ยอด → ไปเดิน Boss Bar) |
| ❌ ไม่ได้ | จบ ไม่ได้ขาย (ต้องเลือกเหตุผล) |
| ☎️ ติดต่อไม่ได้ | โทรไม่ติด/ไม่รับสาย (ต้องเลือกเหตุผล) |

### แอปเตือนเองอัตโนมัติ 🔔
เจ้าไหน**ทักไปแล้วเงียบเกิน 3 วัน** และยังไม่จบ จะขึ้นเตือน:
- **จุดแดงบนแท็บ "ลูกค้า"** บอกจำนวนที่ค้าง
- **การ์ดเตือนบนหน้าเควสต์** แตะเพื่ออัปเดตได้เลย

พออัปเดตครบ จุดแดงหายเอง

> เปลี่ยนจาก 3 วันเป็นเลขอื่นได้ที่ตัวแปร `NUDGE_DAYS` ใน `index.html`

### แท็บ "ลูกค้า" 👥
- **ต้องอัปเดต** — เจ้าที่ค้าง เรียงเงียบนานสุดขึ้นก่อน
- **กำลังตาม** — ทุกเจ้าที่ยังไม่จบ
- **ทั้งหมด** — ประวัติทั้งหมด จัดกลุ่มตามวันที่ทัก

แตะการ์ดไหนก็ได้ → เห็น **timeline ครบว่าเจ้านั้นเดินผ่านอะไรมาบ้าง** และอัปเดตขั้นถัดไปได้

### แท็บ "สรุป" 📋
- **ลูกค้าไปถึงไหนบ้าง** — กราฟแท่งบอกว่าทักไปทั้งหมดกี่เจ้า ค้างอยู่ขั้นไหน **ตายตรงไหนเยอะสุด**
- **ทำไมถึงขายไม่ได้** — นับเหตุผลที่แพ้ เรียงจากมากไปน้อย
- **สัปดาห์นี้คุณทำอะไรไปบ้าง** — ประวัติ 7 วันย้อนหลังของตัวเอง

> 💡 **วิธีอ่าน:** ถ้าตายตอน "ติดต่อไม่ได้" เยอะ = ปัญหาอยู่ที่**คุณภาพรายชื่อ** ต้องหาลิสต์ที่ดีกว่า
> ถ้าตายตอน "เสนอราคาแล้ว" เยอะ = ปัญหาอยู่ที่**ราคา/ข้อเสนอ** คนละเรื่องกันเลย
วันปัจจุบันที่ยังทำไม่ครบยังไม่ถือว่าขาด streak — แต่จะขึ้นเตือนสีส้ม

**Boss Bar** ตั้งต้น 60 ล้าน → 100 ล้าน (แก้ได้ในตั้งค่า)
คำนวณ "ต้องหาลูกค้าใหม่อีกกี่เจ้า" จาก: `ยอดที่เหลือ ÷ (ยอดเฉลี่ยต่อเจ้า × อัตราปิดการขาย)`

## ต่อ Google Sheet — ✅ ทำเสร็จแล้ว (18 ก.ค. 2026)

ระบบเชื่อมกับ Google Sheet เรียบร้อย **ไม่ต้องตั้งค่าอะไรเพิ่ม** เปิดแอปแล้วใช้ได้เลย

| รายการ | ค่า |
|--------|-----|
| Google Sheet | Sales Quest DB (อยู่ใน Google Drive ของเจ้าของ) |
| Apps Script | "Sales Quest API" (standalone) |
| Deployment | Sales Quest v1 · Execute as Me · Access Anyone |
| โทเคนลับ | ตั้งไว้แล้วในทั้ง 2 ฝั่ง (ไม่มีโทเคน = เรียก API ไม่ได้) |

> 🔐 URL และโทเคนตัวจริง **ไม่ได้อยู่ใน repo นี้** (repo เป็น public) เก็บไว้ในไฟล์ `SECRETS-LOCAL.md` ในเครื่องเจ้าของเท่านั้น

**ผลการทดสอบ:** เขียน→อ่านกลับ ผ่าน · ภาษาไทยไม่เพี้ยน · วันที่ถูกฟอร์แมต · ล้างเครื่องแล้วกู้ข้อมูลจากชีทกลับมาครบ · เรียก API โดยไม่มีโทเคนถูกปฏิเสธ

### แจกให้เซลใช้ — ส่ง "ลิงก์ตั้งค่า" ทางไลน์
เพราะ repo เป็น public เราเลย**ไม่ฝัง URL/โทเคนไว้ในไฟล์** ต้องส่งผ่านลิงก์แทน:

1. เปิดแอปในเครื่องเจ้าของ → แท็บ **ตั้งค่า** → กด **🔗 แชร์ลิงก์ตั้งค่าให้เซล**
2. ก๊อปลิงก์ → ส่งเข้าไลน์กลุ่มทีมเซล
3. เซลกดลิงก์ครั้งเดียว → เครื่องจำค่าเชื่อมเอง → ตั้งรหัสผ่านของตัวเอง → ติดตั้งลงหน้าจอโฮม → จบ

หลังจากนั้นเปิดจากไอคอนได้เลย ไม่ต้องใช้ลิงก์อีก

> ⚠️ ลิงก์ตั้งค่ามีกุญแจเข้าข้อมูลอยู่ข้างใน — **ส่งเฉพาะคนในทีม อย่าโพสต์ที่สาธารณะ**
> ลิงก์ธรรมดา (ไม่มี `#setup=`) แชร์ได้ปลอดภัย แต่เปิดแล้วจะไม่เห็นข้อมูลของทีม

### ถ้าต้องแก้โค้ดฝั่ง Sheet ทีหลัง
1. เปิด [Apps Script](https://script.google.com/home/my) → "Sales Quest API"
2. แก้โค้ด → Ctrl+S
3. **Deploy → Manage deployments → ปุ่มดินสอ ✏️ → Version: New version → Deploy**
   > ⚠️ ต้องกดขั้นนี้ทุกครั้ง ไม่งั้น URL เดิมยังรันโค้ดเก่าอยู่
4. ใช้ปุ่มดินสอ (ไม่ใช่ New deployment) จะได้ URL เดิม ไม่ต้องไปแก้ในแอป

### ฟังก์ชันช่วยงาน
ในโค้ดมี `resetSheet()` — เลือกจากดรอปดาวน์แล้วกด Run เพื่อล้างข้อมูลทั้งหมดในชีท (เหลือหัวตาราง) ใช้ตอนอยากเริ่มนับใหม่

## ต่อ Google Sheet ด้วยตัวเอง (ถ้าจะทำชุดใหม่)

ทำครั้งเดียว ~5 นาที

1. เปิด [sheets.new](https://sheets.new) สร้าง Sheet ใหม่ ตั้งชื่อ `Sales Quest DB`
2. เมนู **ส่วนขยาย → Apps Script**
3. ลบโค้ดเดิมทิ้ง แล้วก๊อปเนื้อหาไฟล์ `apps-script/Code.gs` ไปวางทั้งหมด → กดบันทึก 💾
4. กด **ทำให้ใช้งานได้ (Deploy) → การทำให้ใช้งานได้ใหม่**
   - ประเภท: **เว็บแอป**
   - ดำเนินการในฐานะ: **ฉัน**
   - ผู้ที่มีสิทธิ์เข้าถึง: **ทุกคน** ← สำคัญ ไม่งั้นแอปเรียกไม่ได้
5. กดอนุญาตสิทธิ์ (จะมีหน้าเตือน "ไม่ปลอดภัย" → ขั้นสูง → ไปที่ ... เป็นเรื่องปกติของสคริปต์ตัวเอง)
6. ก๊อป **URL เว็บแอป** (ลงท้ายด้วย `/exec`)
7. ในแอป → แท็บ **ตั้งค่า** → วาง URL ลงช่อง Web App URL → **บันทึก + ซิงก์**

เสร็จแล้วมุมขวาบนจะขึ้น "ซิงก์แล้ว" สีเขียว ทำแบบเดียวกันในมือถือของเซลทุกคน (ใช้ URL เดียวกัน)

**การซิงก์ทำงานยังไง:** กดเควสต์ปุ๊บบันทึกลงเครื่องทันที (ใช้ได้แม้เน็ตหลุด) แล้วค่อยส่งขึ้น Sheet เบื้องหลัง + ดึงข้อมูลใหม่ทุก 1 นาทีและทุกครั้งที่เปิดแอป ถ้าเน็ตหลุดจะขึ้น "ค้างส่ง N รายการ" แล้วส่งเองเมื่อเน็ตกลับมา

## ส่งสรุปเข้ากลุ่ม Line อัตโนมัติทุกเช้า 8 โมง

**ตอนนี้:** เข้าแท็บ **สรุป** → กด "📋 ก๊อปส่งเข้า Line" → วางในกลุ่ม (สรุปไม่รวมโหมดส่วนตัวให้อัตโนมัติแล้ว)

**อนาคต — ให้บอทส่งเอง:** ทำต่อจาก Apps Script ตัวเดิมได้เลย ไม่ต้องมี server แยก

1. สร้าง LINE Official Account ที่ [developers.line.biz](https://developers.line.biz) → สร้าง Messaging API channel
2. เอา **Channel Access Token** มา
3. เชิญบอทเข้ากลุ่ม Line ทีมเซล แล้วหา **Group ID** (จาก webhook event)
4. เพิ่มฟังก์ชันใน Apps Script ที่:
   - อ่านข้อมูลจากชีต `entries`
   - **กรองโปรไฟล์ที่ `private: true` ออกก่อนสรุปเสมอ** ← สำคัญ
   - ประกอบข้อความสรุป (สูตรเดียวกับฟังก์ชัน `buildSummary()` ใน `index.html`)
   - ยิง `UrlFetchApp.fetch('https://api.line.me/v2/bot/message/push', ...)`
5. ตั้ง **ทริกเกอร์ตามเวลา** ใน Apps Script → รายวัน 08:00 น. (โซนเวลา Asia/Bangkok)

พร้อมทำเมื่อไหร่ บอก Claude Code ได้เลยว่า _"ช่วยต่อ Line Messaging API ให้ส่งสรุปเข้ากลุ่มทุกเช้า 8 โมง"_

## โครงสร้างไฟล์

```
sales-quest/
├── index.html            ← ตัวแอปทั้งหมด (UI + logic + data layer)
├── manifest.json         ← ข้อมูลแอปสำหรับติดตั้งลงมือถือ (ชื่อ/ไอคอน/สี)
├── sw.js                 ← service worker — ทำให้เปิดได้ตอนไม่มีเน็ต
├── icons/                ← ไอคอนแอปจากโลโก้ MK
├── apps-script/Code.gs   ← backend Google Sheet
└── README.md
```

> เวลาแก้ `index.html` แล้วมือถือยังเห็นของเก่า: เปลี่ยนเลข `VERSION` ใน `sw.js` (เช่น `sq-v1` → `sq-v2`) แล้วอัปใหม่ เครื่องทุกคนจะดึงของใหม่เอง

ใน `index.html` แยกชั้นข้อมูลไว้ชัดที่ object **`Store`** และ **`Sync`**
อนาคตจะเปลี่ยนไปใช้ database จริงหรือ API อื่น แก้แค่ 2 ก้อนนี้ ไม่ต้องแตะ UI

### แก้ค่าที่ใช้บ่อย
- **เพิ่ม/แก้เซล** → ตัวแปร `DEFAULTS.reps` (`private: true` = โหมดส่วนตัว)
- **แก้เควสต์/แต้ม** → ตัวแปร `QUESTS` และ `BONUS`
- **แก้ milestone streak** → ตัวแปร `MILESTONES`

## ข้อควรรู้

- ข้อมูลอยู่ในเครื่อง + Google Sheet ไม่ได้ส่งไปที่อื่น
- ลบรายการ = mark `deleted` ในชีต ไม่ลบแถวจริง (เก็บประวัติไว้ตรวจย้อนหลังได้)
- ดาวน์โหลด CSV ได้จากแท็บตั้งค่า
- "ล้างข้อมูลในเครื่องนี้" ล้างแค่ localStorage — ข้อมูลใน Google Sheet ยังอยู่ ซิงก์กลับมาได้
